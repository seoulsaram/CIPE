
### SW 개발 방법론
- 구조적 방법론 : 절차지향. 하향식
	- 절차적으로 큰 구조 먼저 짜고, 세부 사항을 만들어가는 하향식
- 정보 공학 방법론 : 
	- 기업의 데이터 중심. 기업의 정보 시스템을 체계적으로 개발하기 위한 하나의 전략적 소프트웨어 개발 방법론임. 주로 기업 전체의 업무 흐름, 데이터 흐름, 시스템 흐름을 분석해서 개발함.
- 객체지향 개발 방법론 : 객체간의 관계, 상향식 ⭐️
	- 객체들을 먼저 만들고, 그걸 조립해서 큰 시스템을 만들어나가는 상향식.
	- 클래스/객체 단위를 재사용함.
	- 현실 세계 모델링
	- Java객체, C++ 클래스 등
- CBD 방법론 : 컴포넌트 조합
	- 컴포넌트 단위를 재사용
	- 비즈니스 기능 단위 조립
	- 리액트 컴포넌트, 웹 API 등
- 애자일 방법론 : 변화에 빠르고 유연히 대응

여기서 
하향식 방법론 : 
- 전체 시스템의 큰 구조 -> 세부적 부분으로 설계
- 큰 그림 먼저 보고, 점점 작게 쪼개자 방식
- 추상적이고 전반적인 구조 설계를 먼저 함
- 모듈 계층 구조로 설계
상향식 방법론 :
- 작은 단위의 모듈(기능)들을 먼저 만들고 -> 이들을 조합해 상위 시스템을 구성
- 기초부터 차근차근 쌓아 올리자는 방식
- 재사용 가능한 모듈을 먼저 개발
- 하위 모듈이 우선 -> 테스트도 쉬움
- 상위 구조는 모듈들이 조립되며 점차 형성됨

### 분석 도구
- DFD(Data Flow Diagram) 
	- 시스템 내부의 데이터 흐름과 처리 과정을 시각적으로 표현한 다이어그램
- DD (Data Dictionary) 
	- 시스템에서 사용되는 데이터의 정의와 설명을 모아놓은 문서 또는 도구. 변수명, 자료형, 길이, 단위 등이 포함.
- Minispec 
	- DFD의 프로세스(작업) 하나하나에 대한 간단한 설명서. DFD와 함께 사용되어, 각 기능의 상세 정의를 보완함.
- STD (State Transition Diagram, 상태 전이도) 
	- 시스템이나 객체의 상태 변화를 시각적으로 표현한 다이어그램
- UML (Unified Modeling Language)
	- 객체지향 시스템을 모델링하는 표준화된 언어
	- 클래스 다이어그램, 시퀀스 다이어그램, 유스케이스 다이어그램 등
- ERD (Entity-Relationship Diagram, 개체-관계도)
	- 데이터베이스 설계 시 사용되는 다이어그램
	- 개체(Entity), 속성(Attribute), 관계(Relationship)를 시각화

### CASE 도구
Computer-Aided Software Engineering 도구는 소프트웨어 개발의 전 과정을 컴퓨터로 지원하고 자동화해주는 도구이다. 즉, 소프트웨어 개발을 위한 전용 프로그램.
- 상위 CASE (분석, 설계단에서 사용)
- 하위 CASE (개발 시 사용되는거)

### 객체 지향 분석 모델
- Rumbaugh (럼바우)
	- 객체 모델링 : 객체 다이어그램, 정보 모델링이라고도 하며 시스템에서 요구하는 객체를 찾고 객체들 간의 관계를 정의. 가장 중요하고 선행되어야 함.
	- 동적 모델링 : 상태 다이어그램, 시간의 흐름에 따라 객체들 사이의 제어 흐름, 동작 순서 등의 동적 행위를 표현
	- 기능 모델링 : DFD, 프로세스들의 자료 흐름을 중심으로 처리 과정 표현
- Booch : 미시적 및 거시적 개발
- Jacobson : Use case
- Coad와 Yourdon : ER 다이어그램
- Wirfs-Brok : 분석 설계 구분 없음
